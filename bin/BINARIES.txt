# Pre-built Binaries for Quad Extraction (libQEx)

Windows x64 only. Built with Visual Studio 2022, CMake 3.28.

## Contents

- QEx.dll         - libQEx shared library (links OpenMesh statically)
- QExStatic.lib   - libQEx static library
- OpenMeshCore.lib - OpenMesh core (static)
- OpenMeshTools.lib - OpenMesh tools (static)

## Usage

For Python integration via ctypes, use QEx.dll.
For C++ projects, link against QExStatic.lib + OpenMesh*.lib.

## Build Instructions

If you need to rebuild these binaries:

### Prerequisites
- CMake 3.10+
- Visual Studio 2019 or 2022 with C++ workload
- Git

### Build Steps

1. Create build directory:
   mkdir C:\Slask\libqex_build && cd C:\Slask\libqex_build

2. Clone OpenMesh:
   git clone --recursive https://gitlab.vci.rwth-aachen.de:9000/OpenMesh/OpenMesh.git OpenMesh-src
   cd OpenMesh-src
   git submodule update --init --recursive

3. Build OpenMesh:
   mkdir build && cd build
   cmake .. -G "Visual Studio 17 2022" -A x64 -DCMAKE_INSTALL_PREFIX=../install
   cmake --build . --config Release
   cmake --install . --config Release
   cd ../..

4. Clone libQEx:
   git clone https://github.com/hcebke/libQEx.git libQEx-src
   cd libQEx-src

5. Build libQEx:
   mkdir build && cd build
   cmake .. -G "Visual Studio 17 2022" -A x64 ^
     -DOpenMesh_DIR=C:/Slask/libqex_build/OpenMesh-src/install/share/OpenMesh ^
     -DCMAKE_CXX_FLAGS="-D_USE_MATH_DEFINES"
   cmake --build . --config Release

6. Copy binaries to bin/:
   copy Release\QEx.dll C:\Dev\Corman-Crane\bin\
   copy Release\QExStatic.lib C:\Dev\Corman-Crane\bin\
   copy ..\..\..\OpenMesh-src\install\lib\OpenMeshCore.lib C:\Dev\Corman-Crane\bin\
   copy ..\..\..\OpenMesh-src\install\lib\OpenMeshTools.lib C:\Dev\Corman-Crane\bin\
