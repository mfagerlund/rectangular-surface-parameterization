# Pre-built Binaries for Quad Extraction (libQEx)

**Note:** Binaries are NOT included in the repository (gitignored). They are
automatically downloaded from GitHub Releases on first use.

## Automatic Download

When you run `extract_quads.py`, the appropriate binary for your platform is
downloaded automatically and cached locally:
- Windows: `%LOCALAPPDATA%\rsp\bin\`
- Linux/macOS: `~/.cache/rsp/bin/`

## Manual Download

Pre-built binaries for all platforms are available from GitHub Releases:
https://github.com/mfagerlund/rectangular-surface-parameterization/releases

Download the appropriate binary for your platform:
- `qex_extract-windows-x64` - Windows x64
- `qex_extract-linux-x64` - Linux x64
- `qex_extract-macos-x64` - macOS Intel
- `qex_extract-macos-arm64` - macOS Apple Silicon (M1/M2)

Place the binary in this `bin/` directory or your cache directory.

## Build from Source

If you prefer to build from source:

### Prerequisites
- CMake 3.10+
- C++ compiler with C++11 support
- Git

### Build Steps

1. Clone and build OpenMesh:
   ```bash
   git clone https://gitlab.vci.rwth-aachen.de:9000/OpenMesh/OpenMesh.git
   cd OpenMesh && mkdir build && cd build
   cmake .. -DCMAKE_BUILD_TYPE=Release -DBUILD_APPS=OFF
   cmake --build . --config Release
   cmake --install . --config Release --prefix ../install
   ```

2. Clone and build libQEx:
   ```bash
   git clone https://github.com/hcebke/libQEx.git
   cd libQEx && mkdir build && cd build
   cmake .. -DCMAKE_BUILD_TYPE=Release -DCMAKE_PREFIX_PATH=/path/to/openmesh/install
   cmake --build . --config Release
   ```

3. Copy the `qex_extract` binary to this `bin/` directory.

See `.github/workflows/build-libqex.yml` for platform-specific build details.

## Note

These binaries are for the optional quad extraction step (Stage 6), which is
beyond the scope of the Corman-Crane paper but included for completeness.
The core RSP parameterization (Stages 1-5) works without these binaries.
